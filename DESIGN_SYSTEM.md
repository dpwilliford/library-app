# Design System

## Core Principle

The interface must prioritize clarity, legibility, and structured thinking.

This is a data-rich system used by nontechnical experts. Design must reduce cognitive load, not add to it.

---

# Design Philosophy

## 1. Clarity over decoration
- No unnecessary visual noise
- No decorative elements without purpose
- Every UI element must communicate something

## 2. Information hierarchy is primary
- Users must instantly understand:
  - what they are looking at
  - what they can do
  - what matters most

## 3. Text is first-class content
- This system is text-heavy
- Typography must be highly readable
- Long-form reading must feel comfortable

## 4. Media must be integrated, not dominant
- Images, video, and embeds support understanding
- They should not disrupt reading flow
- They must scale cleanly across devices

## 5. Desktop-first, mobile-safe
- Primary experience: desktop/laptop
- Mobile must remain usable, not identical

---

# Layout System

## Structure

- Top bar (context + identity)
- Left sidebar (navigation)
- Main content column (primary focus)
- Optional right panel (secondary info)

## Content Width

- Optimal reading width: 600–900px
- Avoid full-width text blocks
- Use whitespace to separate sections

---

# Typography

- Sans-serif system font stack
- Clear hierarchy (H1, H2, H3, body)
- Line height: 1.4–1.7

---

# Color System

- Functional colors only
- High contrast
- No arbitrary palette choices

---

# Components

- Buttons: primary, secondary, destructive
- Cards: minimal grouping only
- Tables: structured, readable
- Forms: simple, linear

---

# Media Handling

- Responsive images
- Clean video embeds
- No autoplay

---

# Interaction Design

- Always-visible navigation
- Clear feedback
- Explicit loading states

---

# Role-Based UI

- Student: simple
- Professor: contextual
- Librarian: operational
- Head Librarian: data-heavy
- Admin: system control

---

# Accessibility

- High contrast
- Keyboard support
- Clear focus states

---

# Phase 1 Constraints

- Simple UI only
- Placeholder content
- No over-design

---

# Evolution Rule

Agents must propose and justify design changes before implementation.

---

# Anti-Patterns

- clutter
- inconsistent layouts
- too many colors
- hidden navigation
- premature complexity
