# Phase 1 Checklist

## Goal

Establish a private, working web application with login and role structure.

## Required Features

### Authentication
- login page
- logout
- session persistence

### Roles
- student
- professor
- librarian
- collection-area librarian
- head librarian
- administrator

Use mock users for testing.

### Dashboard Shell
- navigation sidebar
- role-aware UI
- placeholder sections:
  - collection
  - knowledge base
  - recommendations
  - analytics
  - admin

### UI
- minimal
- fast
- mobile-compatible
- readable

### Local Setup
- clear instructions
- install steps
- run command

## Must NOT Include

- catalog import
- AI enrichment
- market search
- recommendation engine
- analytics calculations
- purchasing workflows

## Acceptance Criteria

Phase 1 is complete when:

- app runs locally without errors
- users can log in and log out
- roles change visible UI
- dashboard loads for all roles
- instructions are documented
- no scope creep beyond foundation
