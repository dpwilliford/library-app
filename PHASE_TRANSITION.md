# Phase Transition Rules

## Core Rule

The system may not move to the next phase until the current phase is:
- implemented
- tested
- documented
- reviewed

## Phase 1 → Phase 2

Before starting Phase 2:

Codex must provide:

1. What Phase 1 includes
2. What is complete
3. What is incomplete
4. Known issues
5. How to run and test the app

## Evaluation Questions

- Can the app run locally?
- Can users log in?
- Do roles affect UI?
- Is the structure stable?
- Is anything out of scope?

## Proposal Requirement

Codex must propose Phase 2 before starting it.

Proposal must include:
- scope
- risks
- dependencies
- data model changes

## Human Approval

Phase transitions require human approval.
