# User Roles

## Student

Can:
- search
- view approved reports
- submit recommendations
- explain learning or research need

Cannot:
- approve knowledge claims
- approve purchases
- edit policies

## Professor

Can:
- search
- view reports
- submit recommendations
- connect recommendations to courses, assignments, programs, or research areas

Cannot:
- make final purchase decisions unless explicitly assigned authority

## Librarian

Can:
- review AI-generated claims
- approve or reject metadata enrichment
- review recommendations
- manage assigned collection areas
- create reports
- propose acquisitions to the head librarian

## Collection-Area Librarian

Can:
- oversee assigned collection areas
- monitor usage, gaps, recommendations, and market alerts
- escalate acquisition proposals to the head librarian

## Head Librarian

Can:
- set collection-development targets
- set percentage targets by collection area
- approve, reject, or defer acquisitions
- edit policies
- make final collection-development decisions

## Administrator

Can:
- manage users
- manage permissions
- maintain system configuration
